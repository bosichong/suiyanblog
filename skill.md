# 博文和片语创建 Skill

## 任务描述
静默创建新的博文或片语，保存到对应目录，并使用 git 推送更新。

## 博文创建

### 命令格式
```bash
node createBlog.js <标题> <作者> <标签> <目录> <文件名> <AI标签> -novs
```

**重要**: Agent 必须在命令末尾添加 `-novs` 参数，这样创建文件后不会自动打开 VSCode。

### 参数说明
- `标题`: 博文标题（必填，默认: "博客标题"）
- `作者`: 作者名称（默认: "J.sky"）
- `标签`: 文章标签（默认: "未分类"）
- `目录`: 相对于 md/ 的子目录路径（可选，默认: 空）
- `文件名`: 自定义文件名，不含扩展名（可选，默认: 当前时间戳）
- `AI标签`: AI 生成标识，0、1、2 或 3（默认: 0）

### 文件位置
保存在 `./md/<目录>/<文件名>.md`

### 创建示例
```bash
# 简单创建
node createBlog.js "我的第一篇博文" -novs

# 完整参数创建
node createBlog.js "技术分享" "J.sky" "技术" "tech" "20240319120000" 1 -novs
```

## 片语创建

### 命令格式
```bash
node createThought.js <内容> -novs
```

### 创建示例
```bash
node createThought.js "今天天气真好" -novs
```

## Git 推送流程

创建完文件后，**必须先询问用户是否推送**，用户同意后再执行 git 操作：

### 步骤

1. **查看变更**
```bash
git status
```

2. **添加所有变更**
```bash
git add .
```

3. **生成提交信息并提交**
根据新增/修改的内容生成合适的 commit message：
- 新增文章：`新增文章: <文章标题>`
- 更新文章：`更新文章: <文章标题>`
- 其他更新：`更新: <简要描述>`

```bash
git commit -m "新增文章: 我的文章标题"
```

4. **推送到远程**
```bash
git push
```

### 示例

```bash
# 查看状态
git status

# 添加变更
git add .

# 提交（根据实际情况生成 message）
git commit -m "新增文章: 我的第一个ClawHub技能已发布"

# 推送
git push
```

## 工作流程

1. 根据用户需求确定创建博文还是片语
2. 准备相应的参数
3. 执行创建命令（带 `-novs` 参数）
4. 等待文件创建完成
5. **告知用户创建完成，询问是否推送**
6. **用户同意后**，执行 git 操作：
   - `git status` 查看变更
   - `git add .` 添加变更
   - `git commit -m "..."` 提交（根据内容生成 message）
   - `git push` 推送
7. 确认推送成功

**切记**：创建后必须经过用户确认同意，才能执行 git push！
