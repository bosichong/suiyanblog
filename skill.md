# 博文和片语创建 Skill

## 任务描述
静默创建新的博文或片语，保存到对应目录，并使用 push.py 提交更新。

## 博文创建

### 命令格式
```bash
node createBlog.js <标题> <作者> <标签> <目录> <文件名> <AI标签> -novs
```

**重要**: Agent 必须在命令末尾添加 `-novs` 参数，这样创建文件后不会自动打开 VSCode。

### 参数说明
- `标题`: 博文标题（必填，默认: "博客标题"）
- `作者`: 作者名称（默认: "J.sky"）
- `标签`: 文章标签（默认: "未分类"）
- `目录`: 相对于 md/ 的子目录路径（可选，默认: 空）
- `文件名`: 自定义文件名，不含扩展名（可选，默认: 当前时间戳）
- `AI标签`: AI 生成标识，0 或 1（默认: 0）

### 文件位置
保存在 `./md/<目录>/<文件名>.md`

### 模板内容
```markdown
---
title: '<标题>'
author: '<作者>'
time: '<创建时间>'
tag: '<标签>'
ai_label: <AI标签>
description: '博文的简介'
---

### 可以开始写blog啦(*￣︶￣)'
```

### 创建示例
```bash
# 简单创建
node createBlog.js "我的第一篇博文" -novs

# 完整参数创建
node createBlog.js "技术分享" "J.sky" "技术" "tech" "20240319120000" 1 -novs
```

## 片语创建

### 命令格式
```bash
node createThought.js <内容> -novs
```

**重要**: Agent 必须在命令末尾添加 `-novs` 参数，这样创建文件后不会自动打开 VSCode。

### 参数说明
- `内容`: 片语内容（必填，默认: "我的片语..."）

### 文件位置
保存在 `./md/thoughts/<时间戳>.md`

### 模板内容
```markdown
---
type: thought
time: '<创建时间>'
tag: '闪念'
---

<内容>
```

### 创建示例
```bash
node createThought.js "今天天气真好" -novs
```

## 重要注意事项

### 静默创建
- **Agent 必须在所有创建命令中添加 `-novs` 参数**
- `-novs` 参数会阻止脚本自动打开 VSCode
- 创建文件后直接保存即可，agent 不需要编辑器

### 提交更新（必须询问）
创建完文件后，**必须先询问用户是否推送**，用户同意后再执行：

```bash
python3 push.py
```

push.py 会自动：
1. 检测新增的文章
2. 执行 `git add .`
3. 生成提交信息
4. 执行 `git commit`
5. 执行 `git push`

## 工作流程

1. 根据用户需求确定创建博文还是片语
2. 准备相应的参数
3. 执行创建命令
4. 等待文件创建完成
5. **询问用户是否推送**
6. 用户同意后，执行 `python3 push.py` 提交更新
7. 确认提交成功

## 目录结构
```
md/
├── thoughts/          # 片语目录
│   └── <时间戳>.md
├── <自定义目录>/      # 博文目录
│   └── <文件名>.md
└── <其他>.md          # 根目录下的博文
```
