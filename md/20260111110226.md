今天可真冷啊，不愧是东北的三九天。我干脆窝在家里看书，哪儿也不想去。

刚读到《Unix传奇》里的一段话：“汤普森在一台被弃置的、只有 24KB 内存和 512KB 磁盘的计算机上，用三周时间开发出了 Unix 的原型。”
看到这儿，我忍不住感叹：以前的计算机大神也太厉害了吧！现在的电脑动不动就几十GB内存、上TB硬盘，性能强到离谱，可我们拿着这么强悍的机器，却常常不知道该干点啥。而他们呢，在那么苛刻的条件下，居然写出了影响整个计算机世界的系统。

前几天突然冒出个念头：要不要再学一门新编程语言？找点有意思的事做做。但想来想去，也没决定学哪门。今天看书时又注意到 C 语言和 Unix 的关系——原来正是因为有了 C 语言，才第一次能用高级语言写出整个操作系统。到了 1973 年，Unix 就从汇编语言全面改写成了 C 语言，维护和修改一下子变得容易多了。

这让我想起前阵子在微信读书上看过一本叫《C Primer Plus》的书，好像才看到一半就搁下了。今天翻出来一看，刚好停在第 9 章，讲的是“指针”——地址、内存这些概念，确实是 C 语言里最让人头疼的部分之一。不过既然都翻开了，那就抽空继续往下看吧，慢慢来。

现在这个 AI 编程盛行的时代，重新去啃一本 C 语言的书，听起来可能有点“复古”甚至有点奇怪。但总觉得，C 语言是理解计算机底层绕不开的一道门槛。早些年我也学过一点 C，但那时候同时接触了 Python 和 JavaScript，觉得 C 语言写起来太繁琐，又主要用在系统底层，日常用不上，就慢慢放下了。后来就一直用 JS 和 Python 写点小脚本、搭点小应用，图个方便。

但如果真想搞懂操作系统、内存管理这些底层逻辑，C 语言恐怕还是得学。所以，那本没看完的书，还是接着看吧。

其实前两天我还琢磨着要不要深入学 TypeScript，毕竟想把现在的博客重构一下。结果让 AI 帮我搭了个 TS 开发环境后，反而泄了气——既然 AI 能直接帮我写代码，我干嘛还要花那么多时间去学一套新语法？反正我也会 JavaScript，写点小工具、小页面完全够用。索性就继续“AI 辅助编程”算了，省时省力。

说到底，我也就是个写点小脚本、搭点小玩意儿的普通人。没必要非得追着新技术跑，适合自己节奏就好。

忽的想起那个女程序员的博客还是无法正常打开，看来短期之内是不会恢复正常的访问了，至于其背后的事情如果她不说那就不会有人知道这到底是怎么一回事了。这些年也曾遇到过几个突然关闭的网站，例如：积薪、川流、oba.by等，都是觉得挺好的一个站点。我一般不怎么关注一些个人博客，去年底今年初开始才适当的接触了一些自己喜欢的博客。本来关注的就不多，可别在一个一个的关门大吉了。

外面还在刮风，屋里暖气倒是足。继续看书去了。
