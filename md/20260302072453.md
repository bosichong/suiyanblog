## 起因

我有一块2014买的东芝1T的移动硬盘，好久没有使用了，前一阵子就发现插到USB接口之后，弹出文件管理器一小会就会断开，这可不妙，现在硬盘这么贵，一定要救救她呀。


## 解决连接不上USB

这个USB连接不上和系统无关系Windows下也是这样，看来应该是硬件或是连接线老化出现了问题，首先得排除连接线，京东下单绿联全新usb3.0数据线一条。

今天终于收到了，插上一试果然是数据线老化造成了接触不良的后果，导致移动硬盘供电不足。


## Ubuntu无法挂载

插到usb连接ubuntu提示：

![无法挂载](https://suiyan.cc/assets/images/2025/030201.png)


很明显，这是无法挂载当前的移动硬盘，那咱就解决吧，都这份上了，**救人救到底**。


## 解决方案


一. 添加驱动：



    # NTFS 支持
    sudo apt install ntfs-3g

    # exFAT 支持
    sudo apt install exfat-fuse exfat-utils


二. 移动硬盘手动挂载


	# 创建挂载目录
	sudo mkdir -p /mnt/toshiba
	
	# 手动挂载
	sudo mount -t ntfs-3g /dev/sdb1 /mnt/toshiba
	
	# 现在可以访问了
	cd /mnt/toshiba
	ls



打开

	nautilus /mnt/toshiba
	
	xdg-open /mnt/toshiba


到这里基本上移动硬盘已经可以正常识别和使用了，但是，拔掉USB后，还需要手动挂载。

三. 使用“磁盘”工具（最推荐，简单直观）

Ubuntu 自带了一个叫 **Disks (gnome-disk-utility)** 的工具，可以非常方便地设置自动挂载。

1. 按下 `Super` 键（Windows 键），搜索并打开 **“磁盘” (Disks)**。
    
2. 在左侧列表中找到你的 **Toshiba 移动硬盘**。
    
3. 点击右侧分区图示下方的 **“附加分区选项”**（齿轮图标）。
    
4. 选择 **“编辑挂载选项...” (Edit Mount Options...)**。
    
5. 在弹出的窗口中：
    
    - 关闭 **“用户会话默认值”** 开关。
        
    - 勾选 **“系统启动时挂载”**。
        
    - 在 **“挂载点”** 处填入你喜欢的路径（比如 `/mnt/toshiba`）。
        
    - 点击确定并输入密码。


![无法挂载](https://suiyan.cc/assets/images/2025/QQ20260302-152332.png)


照着图设置啊，基本上到这里就解决问题了，重启机器，随意插拔移动硬盘都可以识别，也可以在资源管理器看到硬盘的盘符。
